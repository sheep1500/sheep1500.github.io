System Design

System Architecture Components

1. API Gateway

역할: API Gateway는 클라이언트로부터 요청을 받아 적절한 마이크로서비스로 전달합니다. 또한, 인증 및 권한 부여, 로드 밸런싱, 캐싱, 요청 제한 등의 기능을 수행합니다.

기술: Kong, NGINX, AWS API Gateway

2. Service Mesh

역할: 마이크로서비스 간의 통신을 관리하고, 보안, 로깅, 트래픽 관리, 서비스 탐색, TLS 암호화와 같은 기능을 제공합니다.

기술: Istio, Linkerd

3. User Service

역할: 사용자 관리 (로그인, 회원가입, 프로필 관리)와 인증/권한 부여 (JWT/OAuth) 기능을 제공합니다.

Database: MySQL (사용자 정보 저장)

4. Subscription Service

역할: 구독 및 결제 처리. 사용자의 구독 상태를 추적하며, 결제 처리 및 상태 갱신을 담당합니다.

결제 시스템: Stripe API, PayPal API

Database: MySQL (구독 기록 저장)

5. Content Management Service

역할: 크리에이터가 업로드한 콘텐츠(동영상, 이미지, 게시글)를 관리합니다. 콘텐츠 메타데이터를 저장하고, 파일은 클라우드 스토리지에 저장합니다.

Database: MySQL (메타데이터 저장)

Storage: AWS S3, Google Cloud Storage

6. News Feed Service

역할: 사용자의 구독에 기반하여 콘텐츠 피드를 구성하고 제공합니다. 구독한 크리에이터의 최신 콘텐츠를 가져와 사용자에게 맞춤형 피드를 제공합니다.

Database: Redis (피드 캐싱), Elasticsearch (피드 검색 및 정렬)

7. Recommendation Service

역할: 사용자의 구독에 기반하여 콘텐츠 피드를 구성하고 제공합니다. 구독한 크리에이터의 최신 콘텐츠를 가져와 사용자에게 맞춤형 피드를 제공합니다.

AI/ML Models: TensorFlow, PyTorch

Database: NoSQL DB (사용자 행동 데이터 저장)

8. Notification Service

역할: 구독자와 크리에이터 간의 실시간 알림을 처리합니다. 새로운 콘텐츠, 결제 실패, 구독 만료 등 중요한 이벤트에 대한 알림을 전송합니다.

기술: Firebase Cloud Messaging (FCM), Apple Push Notification Service (APNS), SendGrid (email)

9. Analytics Service

역할: 사용자의 행동을 분석하여 관리자와 크리에이터에게 통계 정보를 제공합니다. 트래픽, 사용자 참여도, 수익 통계 등을 수집합니다.

기술: Google Analytics, Prometheus (모니터링), Grafana (대시보드 시각화)

10. Payment Service

역할: 사용자의 행동을 분석하여 관리자와 크리에이터에게 통계 정보를 제공합니다. 트래픽, 사용자 참여도, 수익 통계 등을 수집합니다.

기술: Stripe, PayPal, 은행 API (크리에이터 수익 분배)

11. Messaging Service

역할: 구독자와 크리에이터 간의 실시간 메시징 기능을 제공합니다.

기술: WebSockets, RabbitMQ, Redis (실시간 메시지 큐)

12. Admin Service

역할: 관리자용 대시보드 제공 및 사용자, 콘텐츠, 구독 상태를 관리합니다. 시스템 상태를 모니터링하고, 필요 시 개입할 수 있는 도구를 제공합니다.

13. Content Moderation Service

역할: 업로드된 콘텐츠의 규정 준수 여부를 분석하고 모니터링합니다. AI 모델을 사용해 부적절한 콘텐츠를 자동으로 감지합니다.

기술: AI 이미지 분석 서비스 (Google Cloud Vision, AWS Rekognition)

14. Search Service

역할: 크리에이터, 콘텐츠, 카테고리 검색을 처리하며 사용자가 빠르게 콘텐츠를 찾을 수 있도록 지원합니다.

기술: Elasticsearch, Solr

15. Media Service

역할: 동영상 및 이미지 업로드 후 미디어 처리를 수행합니다. 크기 조정, 인코딩, 스트리밍 기능을 제공합니다.

기술: FFmpeg, CloudFront (CDN)

16. Chat(Video) Service

역할: 크리에이터와 구독자 간의 채팅(화상) 기능을 제공합니다.

기술: WebRTC, Messaging Service

Microservices Communication

  • gRPC (Service-to-Service Communication): 각 마이크로서비스 간 고성능 통신을 위해 gRPC를 사용합니다. gRPC는 바이너리 프로토콜로 빠르고 효율적인 데이터 전송을 제공합니다.
  • Message Broker (Asynchronous Communication): RabbitMQ 또는 Apache Kafka를 사용해 비동기 이벤트 기반 통신을 관리합니다. 예를 들어, Notification Service는 새로운 콘텐츠가 게시될 때 RabbitMQ를 통해 알림을 생성할 수 있습니다.
  • Event-Driven Architecture: 모든 중요한 이벤트(콘텐츠 업로드, 결제 성공 등)는 Message Broker를 통해 발행되고 구독됩니다. 이를 통해 느슨한 결합을 유지하며 확장성을 높일 수 있습니다.

System Architecture Diagram Explanation

  1. Client (Web & Mobile App) → API Gateway: 클라이언트는 모든 요청을 API Gateway로 전송합니다. 여기서 인증 및 요청 라우팅이 이루어집니다.
  2. API Gateway ↔ Microservices: API Gateway는 클라이언트 요청을 적절한 마이크로서비스로 전달합니다. User Service, Subscription Service, Content Management Service 등은 API Gateway를 통해 클라이언트 요청을 처리합니다.
  3. Microservices 통신: 각 마이크로서비스는 필요할 때 gRPC를 통해 서로 통신하거나, Message Broker를 통해 비동기적으로 이벤트를 주고받습니다.
  4. AI Content Generation Service, Recommendation Service: AI 기능을 제공하는 서비스는 필요할 때 User Service나 Content Management Service와 통신하여 데이터를 제공하거나 요청을 처리합니다.
  5. Database: 각 마이크로서비스는 자신의 데이터베이스를 사용하여 데이터를 관리합니다. 예: PostgreSQL, Redis, Elasticsearch 등.
