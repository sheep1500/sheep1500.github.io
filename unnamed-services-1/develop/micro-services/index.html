Micro Services

플랫폼을 구성하는 마이크로서비스 목록

1. User Service

  • 기능: 사용자 인증 및 프로필 관리, 권한 부여.
  • 역할: 사용자의 로그인, 회원가입, 프로필 수정, 권한 관리(구독자, 크리에이터, 관리자 등).
  • 필요 기술: JWT 기반 인증, OAuth2 소셜 로그인, 데이터베이스 (사용자 정보 저장).

2. Subscription Service

  • 기능: 구독 및 결제 관리.
  • 역할: 사용자가 크리에이터를 구독하거나 구독을 취소할 수 있는 서비스. 결제 주기 관리 및 구독 상태 업데이트를 처리.
  • 필요 기술: Stripe, PayPal 등의 결제 API, 주기적인 결제 처리 로직, 구독 상태 트래킹.

3. Content Management Service

  • 기능: 크리에이터가 업로드하는 콘텐츠(동영상, 사진, 게시글) 관리.
  • 역할: 크리에이터가 게시물을 업로드하고 관리하는 기능 제공. 업로드된 콘텐츠는 S3나 클라우드 스토리지에 저장.
  • 필요 기술: 파일 저장소 (AWS S3, Google Cloud Storage), 메타데이터 저장 (PostgreSQL, MongoDB).

4. News Feed Service

  • 기능: 구독자에게 최신 콘텐츠 피드 제공.
  • 역할: 구독한 크리에이터의 최신 콘텐츠를 가져와 사용자에게 제공. 피드 정렬 및 필터링.
  • 필요 기술: Redis (캐싱), Elasticsearch (피드 검색/정렬), GraphQL 또는 REST API.

5. News Feed Retrieval Service

  • 기능: 피드를 불러올 때 성능 최적화.
  • 역할: 데이터베이스에서 피드를 효율적으로 검색하고 캐싱하여 구독자에게 신속하게 콘텐츠 제공.
  • 필요 기술: Redis (실시간 캐싱), ElasticSearch (빠른 검색).

6. Notification Service

  • 기능: 알림 전송.
  • 역할: 크리에이터가 새로운 콘텐츠를 게시했거나 구독자가 중요한 이벤트(결제 실패, 구독 만료 등)를 발생했을 때 푸시 알림을 제공.
  • 필요 기술: Firebase Cloud Messaging (FCM), Apple Push Notification Service (APNS), 이메일 전송 (SendGrid, Mailgun).

7. Recommendation Service

  • 기능: 맞춤형 콘텐츠 추천.
  • 역할: 사용자의 나이, 성별, 성향을 기반으로 AI/ML 모델을 통해 맞춤형 콘텐츠를 추천. 사용자 행동 데이터 분석 및 피드백 학습.
  • 필요 기술: ML 모델 (TensorFlow, PyTorch), 사용자 행동 데이터 저장소, 추천 알고리즘 (협업 필터링, 콘텐츠 기반 필터링).

8. AI Content Generation Service

  • 기능: 크리에이터가 AI로 콘텐츠를 쉽게 생성할 수 있도록 지원.
  • 역할: 이미지 생성, 비디오 편집, 텍스트 자동 생성 등 AI 툴 제공. 크리에이터가 자신만의 스타일로 콘텐츠를 쉽게 생성할 수 있게 지원.
  • 필요 기술: OpenAI API (GPT), GANs (이미지 생성), 미디어 처리 도구 (FFmpeg 등).

9. Analytics Service

  • 기능: 데이터 분석 및 통계.
  • 역할: 플랫폼에서의 사용자 활동, 구독자 성장, 콘텐츠 조회수 등의 통계를 수집하고 분석하여 관리자와 크리에이터에게 제공.
  • 필요 기술: Apache Kafka (실시간 데이터 스트리밍), Google Analytics, Prometheus (모니터링).

10. Admin Service

  • 기능: 관리자용 서비스.
  • 역할: 전체 플랫폼 관리 및 모니터링, 사용자 및 콘텐츠 관리, 통계 대시보드 제공.
  • 필요 기술: RBAC(Role-Based Access Control), 데이터베이스 조회 및 업데이트 기능.

11. Payment Service

  • 기능: 결제 처리 및 수익 관리.
  • 역할: 구독 결제, 결제 기록 관리, 크리에이터 수익 분배 처리. 크리에이터가 수익을 인출할 수 있도록 지원.
  • 필요 기술: Stripe, PayPal API, 정산 로직, 은행 계좌 연동.

12. Content Moderation Service

  • 기능: 콘텐츠 관리 및 규정 준수.
  • 역할: 업로드된 콘텐츠를 자동으로 분석해 규정 위반 여부를 감지하고 관리자가 검토할 수 있도록 보고.
  • 필요 기술: AI 기반 이미지 및 텍스트 분석 (클라우드 제공 서비스나 자체 모델).

13. Media Service

  • 기능: 동영상 및 이미지 처리.
  • 역할: 업로드된 동영상 및 이미지의 크기 조정, 인코딩 및 스트리밍 지원. 미디어의 품질을 유지하면서 빠르게 전달.
  • 필요 기술: FFmpeg (미디어 처리), CDN (Content Delivery Network, 예: CloudFront).

14. Messaging Service

  • 기능: 사용자 간 메시징 기능.
  • 역할: 구독자와 크리에이터 간의 실시간 메시지 전송 기능. 필요 시 메시지 저장.
  • 필요 기술: WebSockets, RabbitMQ (메시지 큐), Redis (실시간 메시징).

15. Search Service

  • 기능: 콘텐츠 및 사용자 검색.
  • 역할: 크리에이터, 콘텐츠, 카테고리 등을 검색할 수 있는 기능 제공. 검색 기능 최적화 및 빠른 응답을 위한 인덱싱.
  • 필요 기술: Elasticsearch, Solr.

16. Video Chat Service

  • 기능: 크리에이터와 구독자간 화상 기능.
  • 역할: 구독자와 크리에이터 간의 실시간 화상 기능.
  • 필요 기술: WebRTC, Messaging Service.

마이크로서비스 간 통신 방식

  • API Gateway: 모든 요청을 마이크로서비스에 분배하기 위해 API 게이트웨이를 사용.
  • gRPC: 고성능 내부 서비스 간 통신을 위해 gRPC 사용.
  • Message Broker: RabbitMQ나 Apache Kafka와 같은 메시지 큐를 사용하여 비동기 처리 및 이벤트 기반 아키텍처 구현.
